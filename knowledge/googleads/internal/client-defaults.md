# 甲方默认业务规则（跨市场强制执行）

## ⚠️ 执行优先级

**本文件优先级为 0（最高）。**  
任何市场专属知识文件可覆盖本文件的具体参数，但**不得省略核心漏斗结构**。  
若市场文件未明确说明，一律以本文件为准。

---

## 1. 核心转化漏斗（不可更改）

```
Google Ads 广告曝光
       ↓
   落地页（LP）
       ↓
  社交软件链接点击（LINE / WhatsApp / Telegram / 等）
       ↓
     客户咨询
```

- **入口**：Google Ads（搜索 / GDN / YouTube / PMax）
- **中间层**：落地页，唯一目标是驱动社交软件点击
- **转化动作**：社交软件跳转（非表单、非电话、非下载）
- **咨询渠道**：由市场文件指定（日本默认 LINE，未指定时沿用 LINE）

---

## 2. 目标受众默认值

| 参数 | 默认值 | 可被市场文件覆盖 |
|---|---|---|
| 年龄下限 | **35岁** | ✅ |
| 年龄上限 | 不限 | ✅ |
| 职业画像 | 散户股民 / 个人投资者 | ✅ |
| 设备优先 | 移动端优先 | ✅ |
| 投资经验 | 有一定经验（非零基础） | ✅ |

---

## 3. 落地页强制要求

落地页必须满足以下条件，否则视为未达标：

1. **首屏可见社交软件CTA按钮**（不得藏于折叠下方）
2. **首屏可见合规信息**（注册番号 / 监管机构名称）
3. **主CTA唯一性**：页面核心动作只有一个——点击社交软件按钮
4. **反诈声明**：必须有明确的"非强制入群""非诈骗"类声明
5. **社会证明**：真实用户数据（不得使用未经验证的占位数字）

---

## 4. 广告内容默认约束

- **禁止**：收益承诺、保本保利、具体涨幅数字
- **禁止**：未经验证的"第一名""最强""最准"类绝对化表述
- **必须**：与落地页内容高度一致（广告文案 ↔ LP标题 ↔ CTA）
- **必须**：目标受众年龄定向 ≥ 35岁（Google Ads 受众层设定）

---

## 5. 转化优先级排序

在所有决策冲突时，按以下顺序取舍：

1. **转化率**（LP点击 → 社交软件跳转率）
2. **合规性**（不被Google拒登、符合当地金融监管）
3. **质量得分**（广告相关性、预期CTR、LP体验）
4. **品牌形象**（视觉风格、文案调性）

> 若转化率优化与合规存在冲突，**合规优先**（账号封禁代价远高于转化损失）。  
> 若合规与品牌形象冲突，**合规优先**。

---

## 6. 合规哲学（强制执行）

**核心原则：以转化为目的去合规，而不是为了合规而合规。**

合规本身不是目标，合规是实现转化的工具。每一条合规要求都必须能回答以下问题：

> *"这条合规措施如何帮助我们获得更多转化？"*

合规对转化的具体价值：

| 合规行为 | 对转化的直接作用 |
|---|---|
| 首屏显示金商番号 / 监管牌照 | 消除用户"诈骗"疑虑 → 降低跳出率 → 提高CTA点击 |
| 反诈声明（非强制入群） | 消除35岁+股民最大顾虑 → 提高LINE添加率 |
| 符合Google金融广告政策 | 广告不被拒登 → 流量持续进入漏斗 |
| 合规的风险提示措辞 | 账户不被封禁 → 长期稳定投流 |
| 真实公司信息公示 | 用户搜索公司名时结果正面 → 提高落地页信任度 |

**禁止行为**：
- ❌ 为了"看起来合规"而添加用户看不到的隐藏合规文字
- ❌ 用合规要求作为拒绝转化优化的借口
- ❌ 添加任何会降低转化率但对合规没有实质帮助的元素

**正确决策框架**：
1. 这个合规元素**用户能看到吗**？ → 看不到则转化价值为零
2. 这个合规元素**会降低用户信任还是提高**？ → 提高信任则优先保留
3. 这个合规要求**是Google强制要求还是自我加戏**？ → 自我加戏则删除
