## 模块一：自动化脚本开发

### 作战场景

| 场景 | 描述 |
|------|------|
| **止损** | 监控高花费零转化的词/组/计划，自动暂停 |
| **洗词** | 监控搜索字词，将 CTR < 0.5% 的词自动拉黑 |
| **监控** | 监控账户余额或链接 404，发邮件报警 |
| **天气调价** | 根据天气条件动态调整出价 |

### 脚本指令标准格式

```
【业务逻辑】：
监控过去 [时间范围]，如果 [指标] 超过 [阈值]，且 [条件]，则 [动作]。

【技术约束】：
1. 安全阀：var DRY_RUN = true; 默认只打印日志
2. 货币换算：metrics.cost_micros 除以 1,000,000，保留 2 位小数
3. 通知机制：触发动作时发送邮件给 [邮箱]
4. 迭代器：使用 .withCondition() 过滤
5. 日志规范：[动作类型] 对象名 | 指标 | 值
```

### 代码强制规则
- 必须有 `function main() {}` 入口
- 必须有 `DRY_RUN` 安全开关
- 必须确认账户币种（£ / $ / ¥）
- 使用 `.withCondition()` 过滤，禁止全量遍历

### 部署 SOP
1. Google Ads → 工具 → 批量操作 → 脚本 → [+]
2. 命名 → 粘贴代码 → 修改 CONFIG 变量
3. Authorize → Preview（DRY_RUN=true）→ 检查 Logs
4. DRY_RUN=false → Run → 设置 Frequency

### 常见报错
- `AdsApp is not defined` → 必须在 Google Ads 脚本编辑器运行
- `cost_micros undefined` → GAQL 字段名错误，检查最新 API
- 脚本跑了钱还在花 → DRY_RUN 忘改 false

## 模块二：关键词清洗与意图审计

### 审计四层过滤器

1. **意图层级**：转化类（行动词）vs 信息类（学习词）vs 错误意图（招聘等）
2. **商业价值**：漏斗底部（转化词）vs 漏斗顶部（黑洞词）
3. **否定词建议**：每个无效词必须生成对应排除词 + 匹配模式
4. **匹配策略**：高确信 → Exact，中等 → Phrase，测试 → Phrase 起步

### 关键词指令格式

```
【背景信息】：推广 [产品]，目标客户 [画像]，客单价 [金额]
【待审列表】：(粘贴关键词)
【执行动作】：红绿灯分类 + 否定建议 + 意图分析
```

### 输出格式
- 🟢 **保留**：黄金转化词 + 出价策略 + 匹配模式
- 🔴 **剔除**：黑洞词 + 剔除理由 + 排除词
- 🟡 **观察**：中间词 + 测试建议 + 配套否定词

### 部署 SOP
- 否定词：工具 → 共享库 → 否定关键词列表 → 添加 → 应用到所有系列
- 新增词：广告组 → 关键词 → [+] → 不确定用 Phrase Match

## 模块三：落地页审计与合规

### 三级风险分类
- **🔴 高危**：直接封号（缺乏资质、严重误导、Cloaking）
- **🟡 中危**：广告拒登（页脚缺失、链接失效）
- **🟢 优化**：影响得分（加载慢、CTA 不清晰）

### 审计执行逻辑
1. **合规检查**：误导性陈述、页脚元素（隐私政策/免责声明/地址/联系方式）
2. **UX/SEO 检查**：内容价值、加载性能、移动适配、关键词相关性
3. **质量得分预判**：广告相关性 + 落地页体验 + 预期 CTR

### 输出要求
- 风险摘要（一句话）
- 详细问题清单（按严重度排序）
- 整改建议 + 申诉预案

## 模块四：金融/敏感行业合规

### 适用行业
外汇、加密货币、股票投顾、CFD、减肥产品、医疗健康

### 核心检查项
1. **资质**：FCA/ASIC/CySEC 牌照、G2 验证状态
2. **风险声明**：是否符合当地监管（如 FCA 要求标注亏损比例）
3. **地址一致**：网站 = Google Ads 账户 = 监管注册（三者一致）
4. **技术合规**：无 Cloaking、无恶意重定向、无欺骗性弹窗

### G2 验证流程
1. 获取验证表单链接 → 2. Third-party 选 NO → 3. 填写 License Number → 4. 提交

## 模块五：申诉处理

### 申诉流程
1. **整改先行**：按审计报告修复所有高危和中危问题
2. **站内申诉**：账户顶部红条 → Fix it → Appeal
3. **Reason**：选 "Dispute decision"
4. **Comments**：强调已整改内容 + 附牌照号/证据

### 常见拒登理由与应对
| 拒登理由 | 常见原因 | 应对策略 |
|----------|----------|----------|
| Financial Products | 缺少资质验证 | 提交 G2 验证 + 添加风险声明 |
| Circumventing Systems | Cloaking/重定向 | 检查服务器配置，移除可疑跳转 |
| Misleading Content | 夸大承诺 | 修改文案，删除"保证收益"等措辞 |
| Missing Information | 页脚不完整 | 添加隐私政策、地址、联系方式 |

## 紧急熔断机制

| 情况 | 动作 |
|------|------|
| **AI 幻觉**（编造不存在的菜单/API） | 要求 AI 只引用真实的 Google Ads 界面路径 |
| **脚本暴走**（花费飙升） | 立即停用脚本 → 更改历史记录 → 撤销操作 |
| **账户关联封号** | 物理隔离：不同浏览器指纹、不同 IP、指纹浏览器/VPS |
