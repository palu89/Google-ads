# P5 晚间教育内容

使用前先读 `content-output-enforcement.md`、`story-structure-formula-library.md`、`persona-expression-variance.md`。

本文件只服务以下场景：
- `P5` 晚间教育
- `高表现版`
- 需要更强停留、张力、情绪共鸣和生活切口的晚间教育帖

本文件也是 `P5` 的唯一结构源。
若最终目标是 `合规发布版`，写完本文件结构后，必须再读 `story-mode-switch-sop.md` 做收口。

## 目标
把白天市场造成的混乱、后悔、错判或疲惫，压成一篇更像真人在晚上讲出来的教育帖。

重点不是“讲完整”，而是：
- 先让用户觉得这就是自己
- 再打出一个更深一层的误判或提醒
- 最后留一句能带到明天的判断

## 启动条件
- 用户明确要求 `高表现版`
- 用户明确要求更强张力、反差、停留或故事推进
- 用户明确说不要课堂感、不要标准教学稿、不要整齐解释稿

## 默认长度
- 默认正文区间：`180-320` 字
- 若用户明确要求长稿测试、结构演练或双版本对比，可放宽到 `450` 字以内
- 若用户没有额外说明，不要把 `P5 高表现版` 写成长篇晚课

## 标题公式占位
- 反差误判型：`你以为___，其实真正让你乱的，是___`
- 情绪后劲型：`___之后，最难受的不是___，是___`
- 生活切口型：`___都忙完了，你却还卡在___`
- 晚间提醒型：`收盘以后，最该先停下来的，不是___，是___`

## 推荐骨架
1. 用一个具体而贴身的晚上场景或盘后瞬间切入
2. 迅速指出“表面问题”和“真正问题”不是一回事
3. 打出一个清晰的 `误判 -> 反转` 或 `痛点 -> 提醒`
4. 中段只保留一组核心拆解，不要多组并列教学清单；必要时只留一个刺人的镜面问题
5. 收尾回到人、关系、明天还能用的一句判断

## 强制要求
- 第一段就要让用户感觉“这就是我”或“这件事不对劲”
- 正文必须明显区别于标准晚课，不得写成课堂讲义
- 中段最多保留一组主拆解；若已有主拆解，就不要再叠带编号的问题清单、步骤清单或检查表
- 结尾优先用关系感、提醒感或轻回扣，不要再补第二套教学结构
- 优先从一个具体动作、卡住的念头或盘后瞬间下手，不要先抛抽象心理学术语
- 若确实使用抽象心理术语，必须立刻落到一个当下可见的行为；不能拿术语本身撑段落
- `高表现版` 更适合一记重锤式反问或提醒，不适合再落成带编号的自助清单

## 不符合当前结构的写法
以下任一结构，都不符合当前 `P5 高表现版`：
- 前两段主要停留在泛共情、背景说明或铺垫
- `生活锚点 + 教学拆解 + 自助清单` 的对称结构
- 两组及以上编号清单并列出现
- 主要靠“解释原理 + 给方法”撑全文
- 先抛抽象心理术语，再补一段通用说明
- 用带编号的自助提问当主收口

## 人设要求
- 同一骨架换 persona 时，必须改掉情绪底色、生活锚点、句长节奏、口头纹理
- `馨姊` 这类 persona 可以从生活缝隙切入，但不能只剩“生活感包装”
- `馨姊` 不要写成“先安慰、再解释、再给一串问题”的自助文
- 若换成人设名后还能完全成立，说明区分度不够，需要重写

## 互动收尾公式占位
- 二选一卡点型：`你今晚最卡的是___，还是___？`
- 念头照镜型：`如果把你现在脑子里最吵的那句话写出来，会是___？`
- 生活拉扯型：`你最近更累的，是___，还是___？`
- 明日带走型：`如果明天开盘前只能记一件事，你会留下哪一句？`

## 合格骨架示例
```text
标题：
高开低走之后，最难受的不是亏损，是你一直想把它想明白

正文骨架：
- 晚上场景切入：一个具体动作或瞬间
- 立刻指出真正问题：你不是看不懂，是太急着把今天解释清楚
- 误判 -> 反转：你以为再刷几条消息会安心，其实只是让脑子更吵
- 中段只留一组主拆解：一个行为 + 一个提醒 + 一个镜面问题
- 收尾：回到人和明天，留一句能带走的判断

互动收尾：
你今晚最卡的是不甘心，还是根本停不下来？
```

## 不合格骨架示例
```text
标题：
为什么高开低走会让人越看越乱？

正文骨架：
- 生活场景铺垫
- 解释确认偏误 / 信息过载
- 列 1 / 2 / 3 三个原因
- 再列 1 / 2 / 3 三个问题
- 最后告诉用户把手机放下

问题：
- 太像课堂讲义
- 张力主要来自解释，不来自误判反转
- 编号清单过多
- 收尾还是自助式晚课，不像高表现版
```
