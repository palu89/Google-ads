# Threads 高风险表达替代表达库

本文件用于在某些表达已经进入 `replace` 或 `block` 后，提供可审、可控、可回写的替代表达，不让团队继续靠临场改句子碰运气。

## 一、适用场景

- 某表达已经进入 `replace` 或 `block`
- 同一类风险句式需要统一替换口径
- 不同 persona 需要复用一组更稳妥的表达方案
- 想保留信息价值，但必须换掉风险钩子

## 二、最小判断维度

1. 原表达触发的风险是什么
2. 哪部分信息必须保留
3. 哪部分刺激、承诺或暗示必须删除
4. 替代表达适用于哪些 persona / 栏目
5. 替换后仍需补哪些审稿动作

## 三、最小模板

```text
Threads 高风险表达替代表达：
- 原表达：
- 关联黑名单级别：
- 风险点：
- 必须保留的信息：
- 必须删除的信息：
- 推荐替代表达：
- 适用 persona / 栏目：
- 禁止再出现的近义写法：
- 替换后复核动作：
```

## 四、使用规则

- 替代表达不是把高风险词换成近义词继续刺激
- 若没有能接受的替代表达，应直接删掉该角度，而不是硬保留
- 替换后仍需回到 `threads-review-workflow.md`
- 若替代方案容易被近义改写绕开，应同步 `threads-high-risk-expression-variant-banlist.md`
- 若同一替代方案开始稳定复用，应回写 `threads-post-publish-retrospective.md`
- 若替代后仍连续出问题，应回到 `threads-recurring-high-risk-expression-blacklist.md`
