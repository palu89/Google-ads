# Threads 多周内容节奏压力测试模板

本文件用于在准备拉长内容节奏时，先判断系统能不能扛住，而不是先扩量再失控。

## 一、适用场景

- 准备把节奏从单周扩到多周
- 准备提高周频
- 新阶段想同时推进更多栏目或系列
- 风险恢复后，想重新扩大内容节奏

## 二、测试窗口

- `2w`
- `4w`

## 三、最小判断维度

1. 当前周频和预计周频
2. 审稿承载能力
3. 高风险内容缓冲是否够
4. 系列推进是否会压垮主线
5. 是否保留了回收槽位与应急空间

```text
- 系列承载：
- 回收槽位：
- 过载信号：
- 结论：sustain | tighten | split | pause-expand
- 调整动作：
```

## 五、使用规则

- 若审稿缓冲不足，不做节奏扩张
- 若账号仍处在恢复观察期，不做 `pause-expand` 之外的乐观结论
- 测试结论若为 `tighten`、`split` 或 `pause-expand`，应同步更新周排期与主副线配比
- 若已出现真实过载信号，应转入 `threads-cadence-runaway-rollback-checklist.md`
