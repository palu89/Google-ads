# Threads 账号周会 / 运营周报模板

本文件用于每周做一次账号层同步，避免内容运营只看单篇，不看整周节奏。

它适合用来回答 3 个问题：

- 这周到底做了什么
- 哪些地方开始偏了
- 下周最该收哪条线

## 一、适用场景

- 账号周会
- 每周运营复盘
- 周状态同步
- 周风险整理

## 二、使用顺序

1. 先看 `threads-weekly-content-schedule.md`
2. 再看 `threads-content-queue-ledger.md`
3. 如有发布结果，再参考 `threads-post-publish-retrospective.md`
4. 如涉及 persona 变化，再参考 `persona-lifecycle-switch-sop.md`
5. 如有风险事件，再参考 `threads-risk-incident-log.md`
6. 如当前阶段已有正式目标，再参考 `threads-stage-goal-tracker.md`
7. 如某栏目或某系列明显失真，再参考 `threads-column-lifecycle-assessment.md` 与 `threads-series-continuity-overview.md`
8. 如主线和副线开始失衡，再参考 `threads-account-primary-secondary-track-ratio.md`
9. 如准备拉高下阶段节奏，再参考 `threads-multiweek-content-cadence-stress-test.md`
10. 如库存积压或节奏开始失控，再参考 `threads-content-inventory-aging-rules.md` 与 `threads-cadence-runaway-rollback-checklist.md`
11. 如本周出现人设冲突或高风险栏目重复出问题，再参考 `threads-multi-persona-conflict-arbitration.md` 与 `threads-high-risk-column-cooldown-rules.md`
12. 如异常恢复后准备提频，再参考 `threads-account-content-restart-threshold.md` 与 `threads-content-restart-cadence-escalation-rules.md`
13. 如高风险表达开始绕近义变体，或解封栏目刚回到排期，再参考 `threads-high-risk-expression-variant-banlist.md` 与 `threads-column-release-first-round-schedule.md`
14. 如近义变体已经触发回写，或提频需要明确先后顺序，再参考 `threads-variant-trigger-auto-writeback-rules.md` 与 `threads-cadence-escalation-column-stepup-order.md`

## 三、最小周报维度

1. 本周目标
- 原计划目标是什么
- 完成了多少

2. 发布与排期状态
- 发了几篇
- 卡了几篇
- 哪些还在审稿

3. 本周有效内容
- 哪些篇最值得继续放大

4. 本周失效内容
- 哪些篇不值得继续沿用

5. 当前风险
- 平台风险
- persona 风险
- 节奏风险
- 连续性风险

6. 当前阻塞
- 人设问题
- 排期问题
- 审稿问题
- 证据不足

7. 下周动作
- 保留什么
- 放大什么
- 停掉什么
- 补什么

## 四、最小周报模板

```text
Threads 账号周会 / 运营周报：
- 账号名称：
- 本周目标：
- 完成度：
- 本周已发布：
- 本周待发布：
- 本周 blocked：
- 本周有效内容：
- 本周失效内容：
- 当前 persona 状态：
- 当前主要风险：
- 当前主要阻塞：
- 下周优先动作 1：
- 下周优先动作 2：
- 下周优先动作 3：
```

## 五、使用规则

- 周报不是流水账，要给出明确判断
- 周报必须能反推下周排期
- 若本周有高风险事件，必须明确写出处理动作
- 若周目标连续两周失真，应回到账号基线卡重审
- 若阶段目标连续两个检查点偏离，应回到 `threads-stage-goal-tracker.md` 与 `threads-stage-transition-decision.md`
- 若某栏目或某系列连续两周失真，应进入对应的生命周期或连续性评估
- 若副线连续两周压过主线，应回到 `threads-account-primary-secondary-track-ratio.md`
- 若库存老化明显或审稿积压扩大，应优先减载，不继续扩节奏
- 若同一栏目或同一 persona 冲突连续发生，应优先做仲裁或冷却，不继续硬排
- 若高风险表达连续复发，应先补替代表达库，不继续硬改原句
- 若异常恢复后的重启节奏仍不稳定，不继续提频，应先维持当前级别或回退
- 若升级失败信号接近阈值，应立即参考回退阈值，不继续加速
- 若解封栏目首轮恢复已完成，应先看首轮复盘，再决定是否继续放开
