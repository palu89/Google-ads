# 高表现版转合规发布版 SOP

使用前先读 `story-structure-formula-library.md`、`content-output-enforcement.md`、`threads-copy-compliance.md`。

本文件用于解决一个具体问题：

很多高张力样稿在结构上很强，但直接发会踩到内幕感、收益刺激、强背书或戏剧化过猛的问题。  
这份 SOP 负责把 `高表现版` 收口成保留张力、可审稿、可长期复用的 `合规发布版`。

## 一、适用范围

在以下场景使用：
- 已经先写出一版 `高表现版`
- 需要把内部演练稿改成 Threads 可发布版
- 想保留停留和共鸣，但必须先过审稿与合规链

不适用：
- 纯结构拆解
- 纯内容审稿但没有高表现初稿
- 极短讯提醒型内容，且一开始就直接按 `合规发布版` 写

## 二、转换目标

转换不是“削平”，而是做到三件事：
- 保留原稿最值钱的张力点
- 去掉会踩风险的叙事成分
- 让读者仍然愿意停下来、看完、留言

## 三、输入确认

开始转换前，先确认这 5 项不变：
- 使用的人设
- 当前模块
- 主痛点
- 想保留的核心张力点
- 想让用户带走的判断或提醒

如果这 5 项都说不清，不要开始转换。

## 四、先标出原稿里的高风险张力源

先把原稿里最容易出事的部分圈出来，至少检查这 6 类：
- `内幕感`：我比别人早知道、内部消息、老同事群、提早收到风声
- `强背书`：真实机构、实名头衔、未核实履历、系统内人士
- `收益刺激`：暴涨、翻身、财务自由、买车买房、打脸别人
- `动作诱导`：上车、快跟、错过就没了、马上卡位
- `戏剧化过猛`：像电影桥段、压迫式召回、过强神秘感
- `结果验证过重`：几天大涨、后来都来问我、我果然是对的

## 五、六步转换法

### 1. 保留真正该保留的东西

先挑出原稿里最值钱的 `1-3` 个点：
- 哪个痛点最真实
- 哪个误判最有共鸣
- 哪个反转最有方法价值

只保留这些，不要整篇所有刺激点都想留下来。

### 2. 把“消息优势”改成“结构优势”

优先这样改：
- `我早知道` -> `我比较早把链条想清楚`
- `内部消息` -> `更早看到结构变化`
- `别人还没反应` -> `市场当时还没把这条线看完整`

原则：
- 从“人比别人强”改成“判断框架比别人早一步”

### 3. 把“收益想象”改成“可验证变化”

优先这样改：
- `暴涨` -> `后续资金开始集中反应`
- `翻身` -> `少走很多弯路`
- `买车买房` -> `后来很多人回头才看懂`
- `上车` -> `值得跟踪`

原则：
- 从“结果很爽”改成“市场后来确实补反应”

### 4. 把“人物戏”改成“生活感 + 方法感”

优先这样改：
- 保留一个小生活锚点
- 弱化强神秘设定
- 把人物价值落到“提醒你怎么判断”

例如：
- 不写“加密通讯里传来名单”
- 改成“收盘后回头看，真正该补看的不是大家都在喊的那条线”

### 5. 把“打脸感”改成“事后理解感”

优先这样改：
- `那些笑我的人后来都来问我` -> `后来很多人回头才发现，当时忽略的是这条线`
- `我果然是对的` -> `市场后面慢慢把这层逻辑补齐了`

原则：
- 不做胜利者姿态，改成帮助读者补上认知缺口

### 6. 把结尾收成“提醒”而不是“高潮”

合规发布版结尾优先三选一：
- 一个明天还能用的判断
- 一个适合留言的具体问题
- 一个与 persona 一致的提醒式金句

不要用：
- 过猛的反杀感
- 暗示特殊身份优势
- 把风险说得像确定机会

## 六、四个位置的转换清单

### 标题
- 看有没有过猛的内幕感、打脸感、炫耀感
- 若有，优先改成“反常识 + 提醒”

### 开头
- 保留异常或痛点
- 去掉像电影桥段一样的硬戏剧

### 转折
- 保留误判 -> 反转
- 把“机会煽动”改成“链条解释”

### 收尾
- 保留认知升华或关系感
- 去掉胜利者炫耀和过强结果刺激

## 七、转换后必须复核的 6 件事

1. 现在还是这个 persona 会说的话吗  
2. 原稿最值钱的张力点还在吗  
3. 有没有从“消息优势”改成“结构优势”  
4. 有没有从“收益刺激”改成“市场反应 / 方法提醒”  
5. 现在这版能不能过 `threads-copy-compliance.md`  
6. 现在这版是不是已经明确属于 `合规发布版`

## 八、失败判定

以下任一情况，转换失败，不能当可发布版交付：
- 仍然残留明显内幕感
- 仍然靠收益刺激驱动阅读
- 仍然把未核实人物身份当核心张力
- 结构张力被削平到只剩空话
- 换 persona 后几乎不用改

## 九、最小执行记录

若本次经过本 SOP，内部至少记录：
- 原稿保留的核心张力点
- 原稿删除或弱化的高风险张力点
- 标题如何转换
- 开头如何转换
- 转折如何转换
- 收尾如何转换
- 最终是否通过 `合规发布版` 复核
