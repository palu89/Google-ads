# 故事张力结构公式库

使用前先读 `content-output-enforcement.md`、`content-performance.md` 与 `threads-copy-compliance.md`。

若已经先写出 `高表现版`，并准备转成可发布稿，再读 `story-mode-switch-sop.md`。

本文件不负责决定“要不要发”，而是负责在需要高张力、强停留、强共鸣时，给出可复用的结构公式。

## 一、适用范围

仅在以下情况启用：
- 用户明确要做高张力开头、故事感结构、反差钩子或结构公式演练
- 用户明确要拆解、复用或测试“标题 / 开头 / 转折 / 收尾”的写法
- 某篇内容需要更强停留和情绪推进，但又不能只靠夸张词硬拉流量

以下情况不优先启用：
- 纯审稿
- 纯 persona 治理
- 纯合规问答
- 纯短讯提醒型内容，尤其是 `platform-short` 簇的极短帖

## 二、总原则

- 公式是结构骨架，不是原句复读。
- 先用结构制造张力，再用 persona 把话说得像真人。
- 同一个公式换 persona 时，必须同时调整：情绪底色、生活锚点、句长节奏、互动关系感。
- 不允许把“内幕、未核实情报、强背书、收益验证”当成默认增长捷径。
- 若公式需要借人物权威、信息差或事件升级制造张力，先过 `threads-copy-compliance.md`。

## 三、双模式使用

本公式库分为两层：

### `高表现版`

适用：
- 结构拆解
- 内部演练
- 样稿测试
- 想先把停留、张力和传播感拉满，再做二次收口

特点：
- 允许更强反差、更强异常感、更强误判到反转的推进
- 可以保留更明显的人物戏、情绪链和记忆点
- 但不等于可以直接用于 Threads 可发布版

### `合规发布版`

适用：
- 要交付 `可发布版`
- 要进入 Threads 审稿链
- 要把高张力结构转换成稳定、可长期复用的写法

特点：
- 保留结构张力，但降低内幕感、收益感、强背书感和过度戏剧化
- 默认把“上车、暴涨、内部消息、提早知道”改写成可审稿表达
- 更强调判断、提醒、框架和可带走的方法

### 默认处理规则

- 若用户没有明确说“只做拆解 / 高表现测试”，正式输出默认使用 `合规发布版`
- 若先用 `高表现版` 演练，交付前仍需再过一轮 `合规发布版` 收口
- 若需要把“先拉满张力”的样稿系统性收口成可发布稿，优先使用 `story-mode-switch-sop.md`
- 同一篇不能一半按 `高表现版` 狂拉张力，另一半又假装已经是安全发布稿

## 四、标题公式

| 类型 | 高表现版 | 合规发布版 | 说明 |
|---|---|---|---|
| 反差型 | 大家都在`A`，我今天不`A`，我`B` | 大家都在看`A`，我更在意的是`B` | 合规版收掉过猛反差 |
| 权威型 | `某人/某群体`，比`公开信息源`还快 | 市场都在看表面，我更早在看`另一层线索` | 合规版默认去实名内幕感 |
| 误判型 | 所有人都盯着`A`，真正该看的其实是`B` | 多数人先看`A`，但真正要补看的其实是`B` | 最适合财经结构帖 |
| 结果型 | 那些笑我`A`的人，后来都来问我`B` | 当时很多人没注意，后来资金才慢慢回头看这里 | 合规版去掉“打脸爽感” |
| 认知型 | 真正的`优势/关键`，不是`A`，是`B` | 真正值得记住的，不是`A`，而是`B` | 适合首尾呼应 |

### 使用要求
- 标题优先选 `1` 个主公式，不要混成双标题。
- 如果标题靠“强权威”成立，正文必须补足公开可理解的逻辑链，不允许只靠人物撑住。

## 五、开头公式

| 类型 | 高表现版 | 合规发布版 | 说明 |
|---|---|---|---|
| 异常开头 | `某天/某晚`，我正在`日常场景`，突然发生了`异常信号` | 收盘后回头看今天这盘，我第一个觉得不对的是`某个信号` | 合规版保留异常感，去掉过度戏剧 |
| 极短命令 | 对方只说了`4-8`个字，气氛立刻变了 | 今天只记一句：`某个提醒` | 合规版更适合短帖提醒 |
| 痛点投射 | 如果你也是那种`常见错误/情绪`的人 | 如果你今天也在`某个错判/情绪`里打转 | 最适合 Threads 共鸣起手 |
| 反常识开头 | 明明`常规判断A`，为什么真正该看的是`B` | 明明很多人先看`A`，但我更想提醒你看`B` | 合规版语气更稳 |
| 具体案例 | 用今天`某一档 / 某个盘面结构`，讲一个大多数人都会犯的错 | 就用今天`某个盘面结构`，讲一个很常见的误判 | 最稳的低风险开头 |

### 使用要求
- 开头必须让用户在第一段就知道“这和我有关”或“这事不对劲”。
- 不要连续使用“今天很多人...”这类泛开头，`P5` 仍以 `content-performance.md` 的禁用句式为准。

## 六、转折公式

| 类型 | 高表现版 | 合规发布版 | 说明 |
|---|---|---|---|
| 大众误判 | 先说出普通人最可能想到的答案：`A` | 先点出市场最容易先看的`A` | 先替读者问 |
| 强反转 | 不是`A`，真正要看的是`B` | 光看`A`不够，真正还要补看的是`B` | 合规版降低对抗感 |
| 概念命名 | 把复杂逻辑压成一个好记的新词 | 给这类现象一个便于记忆的说法 | 合规版少一点“神词制造” |
| 链条解释 | 从`事件`推到`受益环节/错判源头` | 从`事件`拆到`该跟踪的链条变化` | 转折后必须补逻辑 |
| 逆势机会 | 正因为`坏现象`发生，才需要改看`另一条线索` | 盘面越乱，越要回头看`真正没被定价的线索` | 合规版不写“上车” |
| 结果验证 | 后面市场真的朝这个方向反应 | 后面资金和盘面开始往这个方向补反应 | 用来收信任，不用来炫耀 |

### 使用要求
- 转折不是靠语气词，是靠“先让读者猜错，再给出更深一层解释”。
- 命名新概念时，要保证它能被后文解释，不要只丢一个酷词。

## 七、收尾公式

| 类型 | 高表现版 | 合规发布版 | 说明 |
|---|---|---|---|
| 情感回收 | 表面讲市场，最后落回人与关系 | 表面讲盘，最后落回普通人最容易乱掉的那个点 | 合规版更适合方法帖 |
| 认知升华 | 真正重要的不是`A`，而是`B` | 以后再遇到这种盘，先记住`B` | 合规版更像提醒 |
| 留问型 | 最后留一个带关系感的具体问题 | 留一个容易回答、贴近正文的问题 | 问法要符合 persona |
| 首尾回扣 | 最后一句回到标题里的那个物件或设定 | 最后一句回扣开头里的误判或提醒 | 适合高完成度内容 |
| 提醒型 | 给一个明天还能用的判断或提醒 | 把今天的案例收成一个明天还能用的判断 | 更适合长期运营内容 |

### 使用要求
- 收尾要么让用户想起自己，要么让用户带走一句判断，不要只是“以上供参考”。
- 故事型内容可以用情感回收；方法型内容优先用提醒型或认知升华。

## 八、人物设定公式

- `亲密关系 + 强经历/强身份 + 寡言少出手 + 看得更深 + 不直接喊单 + 最后带一点情感保护`

适用提醒：
- 亲密权威比远距离专家更容易让人信，但也更容易踩到“虚构背书 / 伪装内幕”风险。
- 若人物身份无法核实，默认弱化为“资深前辈 / 行业老手 / 长期观察者”，不要冒充真实机构背景。

## 九、情绪推进公式

标准顺序：

`异常 -> 紧张 -> 危机/痛点 -> 误判 -> 反转 -> 希望/方法 -> 验证 -> 升华`

使用提醒：
- 不需要每篇都跑满八段。
- 极短帖可以只取：`痛点 -> 反转 -> 提醒`
- 人物故事帖更适合：`异常 -> 紧张 -> 反转 -> 验证 -> 情感回收`

## 十、模式切换规则

从 `高表现版` 切到 `合规发布版` 时，至少做这 6 个动作：
- 把“内幕感”改成“更早看到结构”
- 把“上车 / 暴涨 / 翻身”改成“跟踪 / 反应 / 提醒”
- 把“具体人物强背书”改成“可理解的公开逻辑”
- 把“打脸爽感”改成“事后验证”
- 把“过猛反差”改成“反常识但能解释”
- 把“戏剧高潮”改成“用户可带走的方法或提醒”

## 十一、合规边界

以下元素只允许作为“风险识别案例”或“需改写对象”，不允许默认套入正式成稿公式：
- 体制内内幕
- 未核实采购、合作、战争、政策情报
- 明确暗示“提前知道”
- 具体收益炫耀或短期暴涨验证
- 把“上车、翻身、财务自由”写成近乎确定结果

安全替换建议：
- `内部消息` -> `更早看到结构变化`
- `上车机会` -> `值得跟踪的方向`
- `我早知道` -> `我比较早把链条想清楚`
- `后来暴涨` -> `后续资金开始集中反应`

## 十二、最小执行记录

若本次内容明确调用本公式库，内部至少记录：
- 使用模式：`高表现版 | 合规发布版`
- 使用的标题公式
- 使用的开头公式
- 是否使用转折公式
- 使用的收尾公式
- 是否使用人物设定 / 情绪推进公式
- 是否涉及高风险叙事改写

## 十三、复核问题

写完后至少检查：
1. 这篇是“结构有张力”，还是只是“句子更夸张”？
2. 如果拿掉强身份，逻辑还站得住吗？
3. 这篇有没有把高风险情节包装成增长技巧？
4. 换一个 persona 后，这套公式是否真的出现了细微但明确的变化？
5. 这篇当前到底属于 `高表现版` 还是 `合规发布版`，有没有写到一半混掉？
