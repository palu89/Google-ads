# Threads 内容实验记录模板

本文件用于记录内容运营中的实验，而不是让“试试看”变成无证据的拍脑袋。

适合记录：

- 新开头
- 新 CTA
- 新栏目
- 新 persona 表达
- 新结构顺序

## 一、适用场景

- 想测试新的内容写法
- 想测试新的栏目或主题组合
- 想验证某个 persona 是否适合某类内容
- 想比较两种表达的差异

## 二、使用顺序

1. 先确认不是高风险硬切主 persona
2. 再定义实验假设
3. 再定义变量和观察窗口
4. 实验结束后，决定：
- 推广
- 保留观察
- 停止

## 三、最小记录维度

1. 实验目标
- 想验证什么

2. 实验假设
- 为什么认为这会更好

3. 变量
- 改了什么
- 没改什么

4. 风险边界
- 哪些不能碰

5. 观察结果
- 有什么变化

6. 结论
- 是否值得推广

## 四、最小实验模板

```text
Threads 内容实验记录：
- 实验编号：
- 实验主题：
- 适用 persona：
- 适用栏目 / 模块：
- 实验目标：
- 实验假设：
- 变量：
- 对照项：
- 风险边界：
- 观察窗口：
- 结果摘要：
- 结论：promote | keep_testing | stop
- 下一步动作：
```

## 五、使用规则

- 一个实验尽量只改一个核心变量
- 没有清晰假设，不开实验
- 实验结论必须回流到周报或复盘
- 若实验影响 persona 选择，再回到 `persona-lifecycle-switch-sop.md`
- 若实验进入月度判断，应同步到 `threads-account-monthly-phase-review.md`
