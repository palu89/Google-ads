# P0 人设包装

使用前先读 `content-output-enforcement.md`。

## 目标
让用户先相信“这个人值得听”，再进入后续盘面内容。

## 多人设使用规则
1. 先读 `persona-governance.md` 判断状态、占位符和激活条件，再去 `persona-registry.md` 选定本次使用的主人设
2. 没有明确指定且当前存在 `active` 人设时，才根据身份、年龄阶段、专业背景、目标受众和语气自动匹配
3. 一篇内容只保留一个主人设，避免把两个不同受众、不同身份逻辑的人设混成同一篇
4. 如需做系列账号，先固定每个账号绑定的人设 id，再开始批量创作
5. 写之前先用一句话说明：这个人设是从什么位置看市场、为什么用户会信他、他最该替用户解决什么问题
6. 若该人设不是 `active`，只能输出模板版、测试版或内部草稿，不直接给可发布版本
7. 若当前没有 `active` 人设，不自动代入旧预设；先建新 persona 或先输出结构模板
8. 若当前只有一个 `active` 人设，且用户未明确指定，允许按该 persona 作为默认执行人设

## 占位符模式
- 当用户尚未提供真实头衔、具体年资、机构背景、研究方向时，使用占位符字段，不要脑补。
- 允许在文案中直接保留这些占位符：
  - `{{persona_name}}`
  - `{{role_label}}`
  - `{{authority_signal_1}}`
  - `{{authority_signal_2}}`
  - `{{years_experience}}`
  - `{{market_focus}}`
  - `{{target_audience}}`
  - `{{content_promise}}`
- 如果是对外直接发布版本，优先改写成“不依赖具体数字也能成立”的表达，避免显得像没填完的表单。
- 如果用户明确要“模板版”，就保留占位符原样输出，方便后续替换。
- 若人设仍是 `draft` 或高风险事实字段仍未确认，不把占位符包装成已核实履历。

## 必须回答的三件事
1. 你是谁
2. 你凭什么能讲
3. 你会持续分享什么

## 推荐结构
1. 身份标签：资深投资者、财经内容作者、产业观察者
2. 经历背书：金融年资、研究方向、市场经验、长期关注台湾股市
3. 方法论：`宏观 -> 板块 -> 龙头 -> 个股`
4. 价值观：不神化收益，不盲目喊单，重视风控与节奏
5. 内容承诺：早评、盘中跟踪、盘后复盘、晚间课程

## 常见交付
- 账号简介
- 置顶帖
- 自我介绍
- 个人理念宣言
- 起号前三篇人物包装文案

## 写法要求
- 权威感来自经历和方法，不靠夸张头衔。
- 要有连续叙事：从金融经历走到退休生活，再走到持续看盘与分享。
- 每篇都要埋入“为什么现在愿意讲给大家听”的动机。
- 第一篇亮相时，要同时讲清楚：`我是刚来 Threads`、`我的专业背景`、`我会持续分享什么`
- 新号第一篇结尾必须引导：关注、评论区留言、说出自己最痛的一个问题
- 使用占位符时，优先把不确定信息集中在身份段，不要让整篇每一段都充满变量
- 若是实名或强背书人设，只要仍处于 `draft`，就不要输出像已核实公开发布稿一样的口吻
- 不同人设的差异至少要落在这六层：
  1. 身份逻辑：这个人设为什么会这样看市场
  2. 权威来源：履历、实战、产业经验、教育能力或生活角色
  3. 用户痛点：他面对的是哪一类人、最痛什么
  4. 表达语气：稳重、锋利、陪伴、理性、轻盈或接地气
  5. 内容结构：先讲故事、先讲框架、先讲数据，必须与人设匹配
  6. 互动方式：评论问题要符合该人设最自然的交流方式
- 自检标准：把名字和头衔拿掉后，如果这篇文案仍然像任何一个其他人设都能说，说明人设区分失败
