# Threads 账号阶段退出清单

本文件用于在一个阶段正式结束时做收口，避免“换阶段了，但旧阶段的尾巴没收”。

## 一、适用场景

- 阶段正式结束
- 阶段暂停
- 账号准备换主线
- 需要把旧阶段内容、栏目、风险和资产收干净

## 二、最小退出维度

1. 阶段目标是否完成
2. 当前栏目是否已重新定级
3. 当前系列是否已收口或转交
4. 风险事项是否已关闭或转观察
5. 资产是否已归档或标记回流规则

## 三、最小模板

```text
Threads 账号阶段退出清单：
- 当前阶段：
- 退出原因：
- 阶段结论：
- 栏目处理状态：
- 系列处理状态：
- 风险处理状态：
- 资产处理状态：
- 已完成收口项：
- 未完成收口项：
- 下一阶段入口：
```

## 四、使用规则

- 阶段退出前，先完成 `threads-stage-transition-decision.md`
- 阶段退出时，应同步检查栏目、系列、风险与资产四条线
- 若仍有未完成收口项，不应把当前阶段标记为完全关闭
- 旧阶段完全关闭且新阶段已运行一个检查点后，应补 `threads-account-stage-migration-retrospective.md`
