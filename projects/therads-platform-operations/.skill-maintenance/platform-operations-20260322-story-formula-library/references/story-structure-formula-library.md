# 故事张力结构公式库

使用前先读 `content-output-enforcement.md`、`content-performance.md` 与 `threads-copy-compliance.md`。

本文件不负责决定“要不要发”，而是负责在需要高张力、强停留、强共鸣时，给出可复用的结构公式。

## 一、适用范围

仅在以下情况启用：
- 用户明确要做高张力开头、故事感结构、反差钩子或结构公式演练
- 用户明确要拆解、复用或测试“标题 / 开头 / 转折 / 收尾”的写法
- 某篇内容需要更强停留和情绪推进，但又不能只靠夸张词硬拉流量

以下情况不优先启用：
- 纯审稿
- 纯 persona 治理
- 纯合规问答
- 纯短讯提醒型内容，尤其是 `platform-short` 簇的极短帖

## 二、总原则

- 公式是结构骨架，不是原句复读。
- 先用结构制造张力，再用 persona 把话说得像真人。
- 同一个公式换 persona 时，必须同时调整：情绪底色、生活锚点、句长节奏、互动关系感。
- 不允许把“内幕、未核实情报、强背书、收益验证”当成默认增长捷径。
- 若公式需要借人物权威、信息差或事件升级制造张力，先过 `threads-copy-compliance.md`。

## 三、标题公式

| 类型 | 公式 | 适用场景 | 说明 |
|---|---|---|---|
| 反差型 | 大家都在`A`，我今天不`A`，我`B` | 强停留、强反差 | 适合第一眼抓停留 |
| 权威型 | `某人/某群体`，比`公开信息源`还快 | 故事型、人物带入 | 高风险，默认改成“比市场反应更早” |
| 误判型 | 所有人都盯着`A`，真正该看的其实是`B` | 反共识分析 | 最适合财经结构帖 |
| 结果型 | 那些笑我`A`的人，后来都来问我`B` | 复盘、事后验证 | 不要写成收益炫耀 |
| 认知型 | 真正的`优势/关键`，不是`A`，是`B` | 金句、观点收束 | 适合首尾呼应 |

### 使用要求
- 标题优先选 `1` 个主公式，不要混成双标题。
- 如果标题靠“强权威”成立，正文必须补足公开可理解的逻辑链，不允许只靠人物撑住。

## 四、开头公式

| 类型 | 公式 | 适用场景 | 说明 |
|---|---|---|---|
| 异常开头 | `某天/某晚`，我正在`日常场景`，突然发生了`异常信号` | 故事型、沉浸式内容 | 先日常，后异常 |
| 极短命令 | 对方只说了`4-8`个字，气氛立刻变了 | 高张力人物帖 | 适合制造压迫感 |
| 痛点投射 | 如果你也是那种`常见错误/情绪`的人 | 方法论、教育帖 | 最适合 Threads 共鸣起手 |
| 反常识开头 | 明明`常规判断A`，为什么真正该看的是`B` | 盘面拆解、题材反转 | 适合直接进逻辑 |
| 具体案例 | 用今天`某一档 / 某个盘面结构`，讲一个大多数人都会犯的错 | 教学、P4/P5 | 最稳的低风险开头 |

### 使用要求
- 开头必须让用户在第一段就知道“这和我有关”或“这事不对劲”。
- 不要连续使用“今天很多人...”这类泛开头，`P5` 仍以 `content-performance.md` 的禁用句式为准。

## 五、转折公式

| 类型 | 公式 | 适用场景 | 说明 |
|---|---|---|---|
| 大众误判 | 先说出普通人最可能想到的答案：`A` | 读者代入 | 先替读者问 |
| 强反转 | 不是`A`，真正要看的是`B` | 财经反共识 | 句子要短、狠、准 |
| 概念命名 | 把复杂逻辑压成一个好记的新词 | 传播、记忆点 | 例如“鏟子股”式命名 |
| 链条解释 | 从`事件`推到`受益环节/错判源头` | 方法论、分析帖 | 转折后必须补逻辑 |
| 逆势机会 | 正因为`坏现象`发生，才需要改看`另一条线索` | 结构性机会 | 默认改成“值得跟踪”，少用“上车” |
| 结果验证 | 后面市场真的朝这个方向反应 | 复盘、案例课 | 用来收信任，不用来炫耀 |

### 使用要求
- 转折不是靠语气词，是靠“先让读者猜错，再给出更深一层解释”。
- 命名新概念时，要保证它能被后文解释，不要只丢一个酷词。

## 六、收尾公式

| 类型 | 公式 | 适用场景 | 说明 |
|---|---|---|---|
| 情感回收 | 表面讲市场，最后落回人与关系 | 故事型、人物帖 | 让内容不只像观点输出 |
| 认知升华 | 真正重要的不是`A`，而是`B` | 方法论、金句收口 | 最适合转发传播 |
| 留问型 | 最后留一个带关系感的具体问题 | Threads 互动 | 问法要符合 persona |
| 首尾回扣 | 最后一句回到标题里的那个物件或设定 | 完整闭环 | 适合高完成度内容 |
| 提醒型 | 给一个明天还能用的判断或提醒 | P4/P5、方法帖 | 更适合长期运营内容 |

### 使用要求
- 收尾要么让用户想起自己，要么让用户带走一句判断，不要只是“以上供参考”。
- 故事型内容可以用情感回收；方法型内容优先用提醒型或认知升华。

## 七、人物设定公式

- `亲密关系 + 强经历/强身份 + 寡言少出手 + 看得更深 + 不直接喊单 + 最后带一点情感保护`

适用提醒：
- 亲密权威比远距离专家更容易让人信，但也更容易踩到“虚构背书 / 伪装内幕”风险。
- 若人物身份无法核实，默认弱化为“资深前辈 / 行业老手 / 长期观察者”，不要冒充真实机构背景。

## 八、情绪推进公式

标准顺序：

`异常 -> 紧张 -> 危机/痛点 -> 误判 -> 反转 -> 希望/方法 -> 验证 -> 升华`

使用提醒：
- 不需要每篇都跑满八段。
- 极短帖可以只取：`痛点 -> 反转 -> 提醒`
- 人物故事帖更适合：`异常 -> 紧张 -> 反转 -> 验证 -> 情感回收`

## 九、合规边界

以下元素只允许作为“风险识别案例”或“需改写对象”，不允许默认套入正式成稿公式：
- 体制内内幕
- 未核实采购、合作、战争、政策情报
- 明确暗示“提前知道”
- 具体收益炫耀或短期暴涨验证
- 把“上车、翻身、财务自由”写成近乎确定结果

安全替换建议：
- `内部消息` -> `更早看到结构变化`
- `上车机会` -> `值得跟踪的方向`
- `我早知道` -> `我比较早把链条想清楚`
- `后来暴涨` -> `后续资金开始集中反应`

## 十、最小执行记录

若本次内容明确调用本公式库，内部至少记录：
- 使用的标题公式
- 使用的开头公式
- 是否使用转折公式
- 使用的收尾公式
- 是否使用人物设定 / 情绪推进公式
- 是否涉及高风险叙事改写

## 十一、复核问题

写完后至少检查：
1. 这篇是“结构有张力”，还是只是“句子更夸张”？
2. 如果拿掉强身份，逻辑还站得住吗？
3. 这篇有没有把高风险情节包装成增长技巧？
4. 换一个 persona 后，这套公式是否真的出现了细微但明确的变化？
