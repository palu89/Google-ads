# 赚钱爆帖唯一模板

## 一、主引擎

这类帖最值钱的不是人物，不是温度，不是教你方法。

真正的主引擎只有这个：

- 大家看错了
- 我先看对了
- 我先下手了
- 后来真的涨了
- 之前笑我的人，后来回头问我

一句话：

`赚钱结果 + 可信赚钱路径 + 反大众优越感`

当前 `模版1` 只用于台股/股票场景。
不要再混入业务回款帖、代运营案例、私域成交复盘、仓配利润故事。

## 输入约定

后续默认使用：

```text
模版1
```

补充可选：

```text
人设：<任意标签>
日期：YYYY-MM-DD
事件：<热点事件>
方向：<真正赚钱方向>
```

规则：
- `模版1` 就是唯一核心指令
- 只要出现 `模版1`，就默认使用本模板
- `人设` 与 `日期` 只提供上下文气味，不进入正文
- 未指定 `事件`、`方向` 时，自动随机替换关键事件与赚钱方向
- 指定了 `事件`、`方向` 时，以用户给定为准
- `模版1` 必须先读取 `template-01-engine-map.md`
- 后续只能做槽位替换，不能自由改写体裁和推进方式
- 槽位替换不等于复用旧样稿原句，不得贴句式、贴场景链、贴人物入口

## 二、固定骨架

正文顺序必须固定，不能乱序：

### 1. 强钩子
- 大家都在抢 `A`，我没去
- 那天最热的 `A`，我一股都没碰
- 真正赚钱的，根本不是大家在追的那批

### 2. 私密入口
- 某个比我早半拍的人
- 某个小圈子
- 某份台面下的信息

要求：
- 不能只写“有人告诉我”或“前辈说了一句”
- 必须带一个具体载体：截图、排单、交期表、回款表、后台数据、采购消息、核销数据
- 私密入口要让读者感觉“这条钱路为什么可信”，不是只摆一个人名

### 3. 具体生活场景
- 不能只写“那天”“后来”“有人发来”
- 必须让读者看到一个真实片段：地点、动作、物件、时间感

要求：
- 至少要有两笔具体细节
- 不能只剩一个地名

### 4. 短对话推进
- 对话要把人往前拽
- 不准压缩成“他提醒我不要追”

要求：
- 至少两轮来回
- 至少有一句短、冷、狠的打断
- 不要每句都像总结

### 5. 大众误判
- 所有人都以为机会在 `A`
- 群里、市场、新闻都在喊 `A`
- 正常人的第一反应都会先去追 `A`

### 6. 冷反转
- 错，肉不在那
- 那不是重点
- 真正收钱的，不在前排

### 7. 钉子词
- 给 `B` 起一个记得住的词
- 例如：`验机口 / 排程闸口 / 锁单口`

要求：
- 钉子词必须像行业里真会说的话，或至少像圈内会传的短叫法
- 不准硬造空词，不准用一看就是写作者发明的悬浮概念
- 钉子词最好 `2-5` 个字

### 8. 提前下手
- 我没追最热的
- 我在绿盘先敲了 `B`
- 我提前换到了 `B`

要求：
- 动作必须自然，像真人会说的话
- 不要写成“我去买了液冷泵 / 我去买了封测材料”这种像在买实物的句子
- 更自然的写法是：`我换去敲了两档做___的`、`我把前排砍掉，换到___那边`

### 9. 结果兑现
- 几天后 `B` 开始补涨
- 我买的那几只真的拉起来了
- 前排最热的反而开始震、回落、翻黑

要求：
- 兑现必须写成过程，不能一句“后来真的转了”带过
- 至少写出 `两段变化`：例如第一天没动 / 第二天出量 / 第三天资金补过来
- 必须同时交代：
  - 我这边怎么动了
  - 前排最热的后来怎样了

### 10. 身份反转
- 之前笑我的人后来来问
- 之前劝我追热门的人来问
- 群里最会喊的人反过来问

要求：
- 回头来问的人必须更具体
- 不能只写“有人来问”“之前说我错过风口的人”
- 最好点明：他之前怎么说你，后来怎么回来问

### 11. 金句收口
- 最好赚的，不一定最热
- 钱最后从哪收，比台上谁最吵更重要
- 会赚钱的人，不是追最亮那盏灯的人

## 二点五、顺序硬规则

- 不允许开头第一句就写“我赚了多少”
- 不允许先写结果兑现，再倒回来补前因
- 不允许省略私密入口，直接跳到行业解释
- 不允许省略生活场景和短对话，直接写成压缩摘要
- 不允许省略大众误判，直接给正确答案
- 不允许把身份反转和金句收口提前
- 不允许带任何外壳标签、审稿标签、分节标签

## 四、逻辑约束

- 热点是 AI 伺服器，真正赚钱方向就要和机柜链、连接链、出货链有关
- 热点是记忆体，真正赚钱方向就要和封测、载板、材料、设备链有关
- 人物是经济智库顾问，信息容器更适合简报、排单、产业群
- 人物是供应链前辈，信息容器更适合交期、采购、缺料、订单
- 每次至少随机替换 6 个槽位，且要成套替换，不能只换一两个表层词
- 可替换的是外壳槽位，不可替换的是发动机、推进顺序、打脸强度、过程型兑现和回头来问闭环
- 必须避开 `template-01-engine-map.md` 中列出的绝对禁止复用项
- 每次至少替换 8 项关键槽位，且必须成套替换
- 不准用过于宽泛的大类名词直接充当真正赚钱方向，例如：`封测材料 / 散热链 / 后排供应链 / 液冷`
- 真正赚钱方向必须具体到一个足够像交易对象、合同对象、订单对象或收费对象的层级
- 私密入口必须具体到一份表、一张图、一条消息或一个能落地的证据
- 回头来问的人必须和前面的质疑形成对应，不能空泛
- 不能写成 6-7 句的摘要短稿
- 至少要让读者看到：一个场景、两轮短对话、一个过程型兑现

## 五、失败示例特征

以下任一出现，都说明没按 `模版1` 写：
- 开头先报回款、净利、预付款、收益率
- 一开头就给结论，没有“大家都在抢什么”
- 中间只有解释，没有“我先去买了什么”
- 结果兑现很弱，只说“后来被看见了”
- 没有人回来问
- 真正赚钱方向是泛词，不像真实交易对象
- 钉子词太悬浮，一看就像硬造词
- 私密入口太薄，只剩“有人说了一句”
- 回头来问的人太空，只写“有人来问”
- 没有具体生活场景，只有一句地点说明
- 对话被压缩成一句总结，没有两轮推进
- 正文短得像摘要或槽位速记
- 文末带 `互动收尾`、`审稿结论` 之类壳子
- 只保留了结构词，没保留场景密度、人物厚度和对话推进
- 把“槽位替换”写成了“另一种新帖子”
- 和原帖或旧样稿在人物、场景、短句、节奏上过度相似
