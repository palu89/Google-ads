# 模版1 结构地图

这份 reference 不是样稿库。

这里不保存任何“成功稿正文”，也不提供任何可直接贴近复用的句子。

它只保留三类东西：

- `模版1` 必须保留的功能位
- `模版1` 可替换的槽位
- `模版1` 绝对不能复用的旧稿特征

## 一、模版1 只保留这些功能位

`模版1` 只能保留下面这些功能位，顺序不能乱：

1. 强钩子
2. 私密入口
3. 具体生活场景
4. 短对话推进
5. 大众误判
6. 冷反转
7. 钉子词
8. 提前下手
9. 过程型兑现
10. 身份反转
11. 金句收口

说明：

- 保留的是功能，不是句子
- 保留的是推进，不是现成段落
- 保留的是发动机，不是某一篇成功稿

## 二、全部可替换槽位

每次生成至少替换 8 项，且以下 8 项必须全部替换：

1. 关系人物
2. 信息容器
3. 权威身份
4. 开场场景
5. 热点事件
6. 真正赚钱方向
7. 钉子词
8. 结果兑现方式

还应优先继续替换：

9. 大众先看方向
10. 可信证据
11. 提前下手方式
12. 回头来问的人

### 1. 关系人物
- 老哥
- 表哥
- 学长
- 老板
- 前辈
- 做产业的朋友

### 2. 信息容器
- 采购群
- 供应链群
- 小饭局
- 排单表
- 交货表
- 后台截图

### 3. 权威身份
- 经济智库顾问
- 供应链老手
- 采购前辈
- 做行业很多年的朋友
- 退场老手

### 4. 开场场景
- 便利店外面等车
- 停车场
- 回家路上
- 地下车库
- 店里吃到一半
- 深夜还没睡
- 办公室外抽烟区

### 5. 热点事件
- AI 伺服器
- 記憶體
- 重電
- 散熱
- PCB
- 航運

### 6. 大众先看方向
- 最热本体
- 龙头股
- 台上最亮那批
- 一开盘最会冲的那批
- 热搜中央那批

### 7. 真正赚钱方向
- ABF载板
- 测试介面
- 探针卡
- 清洗耗材
- 电源模组
- 液冷泵

要求：
- 不准用宽泛大类词直接充当真正赚钱方向
- 例如不要只写：`封测材料 / 散热链 / 后排供应链 / 液冷`
- 必须具体到交易对象、订单对象、收费对象或明确定义的链条节点

### 8. 钉子词
- 验机口
- 排程闸口
- 锁单口

要求：
- 钉子词必须短、像圈内会说的话
- 不准写成悬浮概念词

### 9. 可信证据
- 排单
- 交期延后
- 出货表
- 采购在追
- 上游先缺
- 排程延后名单

要求：
- 私密入口至少落到一个具体证据，不准只有“某人说了一句”

### 10. 提前下手方式
- 绿盘慢慢敲
- 分批换进去
- 先砍热门，换到后排

### 11. 结果兑现方式
- 三天拉快两成
- 放量创波段新高
- 资金开始补涨
- 龙头震了，后排接棒

要求：
- 兑现必须带过程，不准一句带过
- 最少交代两段变化

### 12. 回头来问的人
- 之前笑我的人
- 之前叫我追热门的人
- 群里最会喊的人
- 之前说我错过风口的人

要求：
- 回头来问的人要尽量具体
- 最好能和前文质疑形成对应

## 三、逻辑约束

- 热点、真正赚钱方向、可信证据、兑现方式必须互相对得上
- 热点如果是硬件链，真正赚钱方向必须来自硬件链的后排收钱口
- 不能只换热点，不换人物、场景、信息源
- 不能只换名词，不换推进关系
- 私密入口必须具体
- 兑现必须过程化
- 回头来问必须有对应关系

## 四、绝对禁止复用

后续生成时，绝对不要复用这些旧稿特征：

- 我爸的「老同事群」
- 比總統府即時新聞還快
- 現在回家
- 荷姆茲海峽
- 鏟子股
- 我哥那個「老鄉群」，比法人晨報還早半拍
- 先別追伺服器
- 真正的肉，在收費口
- 過路費股
- 慢，才輪得到你
- 散戶最愛搶台上最亮那盞燈
- 其實也沒多神

也不要整段复用：

- 火锅店 -> 去他家 -> 三台屏幕 -> 老乡群 -> 收费口 -> 三档名单 -> 三天转强 -> 朋友回头问
- “你到底怎么先看到的？” -> “其实也没多神” -> “钱最后从哪收”

## 五、当前线程污染规则

如果当前会话里已经出现：

- 原帖全文
- 旧成功稿全文
- 任何接近原帖的整段示例

则生成时必须额外避开：

- 同类人物入口
- 同类开场场景
- 同类钉子词
- 同类回头来问句型

一句话：

`保留功能位，强制大换皮。`
