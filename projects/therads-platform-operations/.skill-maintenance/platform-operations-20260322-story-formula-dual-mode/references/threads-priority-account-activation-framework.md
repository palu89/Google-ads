# Threads 优先启动账户框架

本文件用于从 `10` 个账户里选出首批 `2-3` 个重点运营账号。

## 一、原则

- 先集中打透，不平均铺开
- 首批账号必须互补，不互相打架
- 先选“最能形成内容闭环”的账号

## 二、推荐首批组合

推荐优先：

1. `主品牌号`
2. `1` 个核心垂类号
3. `1` 个核心人设号

原因：

- 主品牌号负责总信任
- 垂类号负责专业深耕
- 人设号负责温度、叙事和互动

## 三、评估维度

| 维度 | 问题 | 分值 |
| --- | --- | --- |
| 定位清晰度 | 该号的存在理由是否一句话能说清 | 1-5 |
| persona 就绪度 | 当前 persona 是否已 ready | 1-5 |
| 内容库存 | 是否有连续 `2-4` 周内容可支撑 | 1-5 |
| 钩子适配度 | 是否容易形成稳定钩子 | 1-5 |
| 风险可控度 | 是否容易踩平台或信任风险 | 1-5 |
| 协同价值 | 是否能和其他号形成转发 / 引流 | 1-5 |

总分高者优先。

## 四、输出模板

- 当前判断
- 推荐优先启动账号
- 不建议首批启动的账号
- 每个优先账号的存在理由
- 三者之间的协同关系
- 首批两周节奏建议
