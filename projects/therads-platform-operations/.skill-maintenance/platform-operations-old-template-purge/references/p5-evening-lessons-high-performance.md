# P5 晚间教育内容

使用前先读 `content-output-enforcement.md`、`story-structure-formula-library.md`、`persona-expression-variance.md`。

本文件只服务以下场景：
- `P5` 晚间教育
- `高表现版`
- 需要更强停留、张力、情绪共鸣和生活切口的晚间教育帖

本文件也是 `P5` 的唯一结构源。
若最终目标是 `合规发布版`，写完本文件结构后，必须再读 `story-mode-switch-sop.md` 做收口。

## 目标
把白天市场造成的混乱、后悔、错判或疲惫，压成一篇更像真人在晚上讲出来的教育帖。

重点不是“讲完整”，而是：
- 先让用户觉得这就是自己
- 再打出一个更深一层的误判或提醒
- 最后留一句能带到明天的判断

## 启动条件
- 用户明确要求 `高表现版`
- 用户明确要求更强张力、反差、停留或故事推进
- 用户明确说不要课堂感、不要标准教学稿、不要整齐解释稿

## 推荐骨架
1. 用一个具体而贴身的晚上场景或盘后瞬间切入
2. 迅速指出“表面问题”和“真正问题”不是一回事
3. 打出一个清晰的 `误判 -> 反转` 或 `痛点 -> 提醒`
4. 中段只保留一组核心拆解，不要多组并列教学清单；必要时只留一个刺人的镜面问题
5. 收尾回到人、关系、明天还能用的一句判断

## 强制要求
- 第一段就要让用户感觉“这就是我”或“这件事不对劲”
- 正文必须明显区别于标准晚课，不得写成课堂讲义
- 中段最多保留一组主拆解；若已有主拆解，就不要再叠带编号的问题清单、步骤清单或检查表
- 结尾优先用关系感、提醒感或轻回扣，不要再补第二套教学结构
- 优先从一个具体动作、卡住的念头或盘后瞬间下手，不要先抛抽象心理学术语
- 若确实使用抽象心理术语，必须立刻落到一个当下可见的行为；不能拿术语本身撑段落
- `高表现版` 更适合一记重锤式反问或提醒，不适合再落成带编号的自助清单

## 不符合当前结构的写法
以下任一结构，都不符合当前 `P5 高表现版`：
- 前两段主要停留在泛共情、背景说明或铺垫
- `生活锚点 + 教学拆解 + 自助清单` 的对称结构
- 两组及以上编号清单并列出现
- 主要靠“解释原理 + 给方法”撑全文
- 先抛抽象心理术语，再补一段通用说明
- 用带编号的自助提问当主收口

## 人设要求
- 同一骨架换 persona 时，必须改掉情绪底色、生活锚点、句长节奏、口头纹理
- `馨姊` 这类 persona 可以从生活缝隙切入，但不能只剩“生活感包装”
- `馨姊` 不要写成“先安慰、再解释、再给一串问题”的自助文
- 若换成人设名后还能完全成立，说明区分度不够，需要重写
