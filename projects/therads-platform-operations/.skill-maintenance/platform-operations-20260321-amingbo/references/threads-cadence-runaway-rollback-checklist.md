# Threads 节奏失控回退清单

本文件用于在内容节奏已经开始失控时，快速回退到可控状态，而不是继续硬撑。

## 一、适用场景

- 周频扩张后开始积压
- 审稿跟不上
- 高风险内容缓冲消失
- 系列、栏目和副线一起抢资源
- 明显出现“越做越乱”的信号

## 二、最小失控信号

1. 队列积压明显
2. `review` / `revise` 堵塞
3. 主线推进被副线挤压
4. 高风险内容没有缓冲
5. 多 persona 切换开始混乱

## 三、最小模板

```text
Threads 节奏失控回退：
- 当前阶段：
- 当前失控信号：
- 当前过载来源：
- 立即暂停项：
- 立即保留项：
- 需要削减的副线 / 系列 / 栏目：
- 回退后目标周频：
- 回退后主线 / 副线配比：
- 退出回退条件：
- 下一次检查点：
```

## 四、使用规则

- 先减载，再解释
- 回退动作应同步更新 `threads-weekly-content-schedule.md` 与 `threads-content-queue-ledger.md`
- 若失控来自多周扩张失败，应同步回写 `threads-multiweek-content-cadence-stress-test.md`
- 若失控来自多 persona 轮值混乱，应同步回写 `threads-multi-persona-rotation-schedule.md`
