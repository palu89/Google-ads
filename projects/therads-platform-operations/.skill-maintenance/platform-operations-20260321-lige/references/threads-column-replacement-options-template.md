# Threads 栏目替代方案模板

本文件用于在某个栏目进入冷却、冻结或临时退场时，快速找到能承接同一阶段目标的替代栏目，避免排期直接断层。

## 一、适用场景

- 某栏目进入 `cooldown` 或 `freeze`
- 某栏目需要暂时退场，但当前周目标不能空窗
- 想用低风险栏目承接原栏目任务
- 需要判断替代是临时补位还是阶段迁移

## 二、最小判断维度

1. 原栏目承担的阶段目标是什么
2. 替代栏目能否承接同类目标
3. 风险是否明显下降
4. 是否影响 persona 与内容支柱映射
5. 替代结束后如何回切或常驻

## 三、最小模板

```text
Threads 栏目替代方案：
- 原栏目：
- 触发原因：
- 当前阶段目标：
- 默认 persona：
- 候选替代栏目：
- 推荐替代栏目：
- 风险下降依据：
- 是否临时补位：
- 生效周期：
- 回切 / 常驻条件：
- 需同步更新的文件：
```

## 四、使用规则

- 栏目替代不是换个名字继续发同样高风险内容
- 替代栏目必须说明承接的是哪个目标，而不是只看短期流量
- 若替代方案影响内容支柱或 persona 边界，应同步 `persona-content-pillar-map.md`
- 替代方案进入周排期前，应同步 `threads-weekly-content-schedule.md`
- 若原栏目准备恢复，应同步 `threads-column-cooldown-release-checklist.md`
- 若没有可接受替代方案，应回到 `threads-column-lifecycle-assessment.md`
