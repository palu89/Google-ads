# Threads 账号阶段迁移回顾模板

本文件用于在阶段切换后回顾迁移质量，而不是只看“新阶段已经开始了”。

## 一、适用场景

- 新阶段已经运行 1 到 2 个检查点
- 需要确认旧阶段尾巴有没有带进来
- 需要判断这次迁移是稳定、观察还是要重做

## 二、最小判断维度

1. 来源阶段和目标阶段是否真的切清
2. 哪些资产、栏目、系列被保留
3. 哪些内容包袱被带进了新阶段
4. 新阶段当前已经稳定了什么
5. 哪些地方仍在摩擦或失真

## 三、最小模板

```text
Threads 账号阶段迁移回顾：
- 来源阶段：
- 目标阶段：
- 迁移时间：
- 保留资产：
- 放弃资产：
- 迁移冲突：
- 当前稳定项：
- 当前未稳定项：
- 需要修正的主线 / 配比 / 节奏：
- 回顾结论：stable | watch | rework
- 下一步动作：
```

## 四、使用规则

- 迁移回顾应发生在新阶段运行至少一个检查点之后
- 若结论为 `rework`，应回到 `threads-stage-transition-decision.md`
- 若结论为 `stable`，应同步更新 `threads-stage-goal-tracker.md`
