# Threads 内容资产归档规则

本文件用于给旧内容、历史批次和阶段性素材做分层归档，而不是一直堆在活跃层。

## 一、归档对象

- 已发布但不再适合复用的内容
- 历史批次草稿
- 阶段收口后的旧模板
- 风险事件相关临时素材

## 二、归档层级

- `active`：仍可直接使用
- `reference`：可参考，但不直接复用
- `archive`：历史留存
- `blocked`：因风险或过时暂不使用

## 三、最小判断维度

1. 是否仍可直接复用
2. 是否只适合作参考
3. 是否已失效或过时
4. 是否因风险被封存

## 四、最小模板

```text
Threads 内容资产归档：
- 资产名称：
- 资产类型：
- 当前层级：active | reference | archive | blocked
- 归档原因：
- 是否可回用：
- 关联 persona / 栏目：
- 后续引用规则：
```

## 五、使用规则

- 不把明显过时的内容继续留在活跃层
- `blocked` 资产不得无说明回流
- 阶段收口后应做一次归档整理
- 若 `reference` 或 `archive` 资产准备回流，应补一份 `threads-archived-asset-reactivation-rules.md`
