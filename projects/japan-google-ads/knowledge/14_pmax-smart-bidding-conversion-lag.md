## 一、核心认知：日本金融广告的自动化投放陷阱

**最重要的底层逻辑**：日本证券开户场景下，广告系统优化的"表面转化"与"真实价值转化"存在巨大鸿沟。

```
表面转化（前端）：点击注册按钮、提交邮箱、完成开户申请
              ↕ 约50-80%的表面转化是无效客户（低质填表/激励流量）

真实价值转化（后端）：首次入金5万日元以上、实际交易
```

**核心结论**：任何出价策略的优化目标，必须锚定**后端深度转化**，而非前端浅层事件。

## 二、Performance Max (PMax) vs 传统 Search + GDN

### 2.1 CPA差异数据（三来源存在冲突，已标注）

| 投放模式 | CPA相对水平 | 数据来源 | 备注 |
|:---|:---|:---|:---|
| **Search（精准关键词）** | 100%（基准） | 🟢 实战基准 | 高意图，后端LTV最高 |
| **Search + RLSA** | 70%（最低）| 🔴 | 再营销叠加 |
| **PMax（浅层转化目标）** | 低20%-40%（**虚低**）| 🔴 Google来源 | ⚠️ 陷阱：系统寻找廉价填表，80%是无效客户 |
| **PMax（深层入金转化目标）** | Search的120%-180% | 🔴 GPT来源 | 若目标正确，CPA更高但质量好 |
| **GDN（展示广告）** | 200%以上 | 🔴 | 冷流量，直接转化最难 |

> ⚠️ **关键冲突解读**：
> - Google来源说"PMax CPA比Search低20%-40%"——指的是**浅层表单提交**目标，本质是低质廉价流量的汇聚
> - GPT来源说"PMax=Search的1.2-1.8倍"——指的是**质量调整后**的真实CPA
> - **操作结论一致**：PMax用浅层目标→CPA表面低，但后端价值极差；PMax用深层入金目标→CPA更高，但投资回报更合理

### 2.2 PMax "前端CPA虚低"陷阱机制

```
失败模式（Failure Mode）：
PMax目标 = "提交邮箱" / "点击注册按钮"
         ↓
算法迅速收敛到全网最便宜的流量池
         ↓
大量来源：移动端游戏内激励广告 / 低质网站 / 机器人填表
         ↓
表面开户数极低CPA ✓ 但后端入金率极差 ✗ （高达80%无效线索）
```

**正确使用PMax的条件**：
1. 转化目标必须设为**首次入金完了（初回入金完了）**
2. 使用 Offline Conversion Tracking (OCT) 将入金数据回传Google
3. 账户已积累足够转化历史，算法有高质量样本可学习

### 2.3 PMax在成熟账户中的正确定位

| 定位 | 作用 |
|:---|:---|
| **核心获客** | ❌ 不适合（精准关键词Search更优） |
| **规模扩展/拓新** | ✅ 适合（在核心Search跑稳后用于放量） |
| **品牌曝光** | ✅ 适合 |

**推荐预算结构**（成熟账户）：

| 渠道 | 预算占比 |
|:---|:---|
| Search（核心关键词） | 70% |
| PMax（扩量） | 20% |
| 再营销 | 10% |

## 三、Target CPA (tCPA) 学习期与数据密度要求

### 3.1 转化量门槛（三来源综合）

| 阶段 | 转化量要求（过去30天）| 可信度 | 说明 |
|:---|:---|:---|:---|
| **官方最低门槛** | 15次 | 🟢 Google官方 | 算法开始学习的最低条件 |
| **实际稳定运行** | 30次 | 🟡 行业基准 | 多数账户实用下限 |
| **日本金融推荐门槛** | **50-100次高质量入金转化** | 🔴 实战经验 | 竞争激烈的NISA/証券口座词包下的实际需求 |

> **为什么日本金融需要更多转化**：证券开户有极大的用户决策摩擦（My Number Card/本人確認書類准备），15次转化远不足以稳定模型，CPA会随大盘剧烈波动。

### 3.2 学习期时长

| 场景 | 学习期时长 |
|:---|:---|
| 转化量充足（50+/月） | 7-14天 🟡 |
| 转化量不足 | 3-4周，甚至持续波动 🔴 |

### 3.3 学习期重置（最容易踩的坑）

以下操作会直接触发算法重新进入学习期（3-7天波动）：

| 触发操作 | 影响程度 |
|:---|:---|
| 单次调整tCPA目标值 **>15%-20%** | 🔴 直接重置 |
| 大幅修改Campaign预算（超过50%） | 🔴 直接重置 |
| 修改转化目标 | 🔴 直接重置 |
| 添加/删除大量关键词 | 🟡 可能影响 |

> ⚠️ **日系甲方常见失误**：极度喜欢每天看报表调价（每次调5%-10%），这种频繁微调会让账户长期处于学习状态，永远无法发挥Smart Bidding真实效能。**策略：一次调整到位，至少等待7天观察效果再调整。**

## 四、转化延迟 (Conversion Lag/Delay)

### 4.1 日本券商典型转化延迟数据

| 漏斗阶段 | 延迟时间 | 可信度 | 说明 |
|:---|:---|:---|:---|
| 点击广告 → 提交开户申请 | **1-3天** | 🔴 | 用户需回家找身份证件/My Number Card |
| 申请 → 账户审核通过 | **3-7天** | 🔴 | AML反洗钱+eKYC流程（即使SBI/乐天也需数天）|
| 开户 → 首次入金/交易 | **7-21天，甚至30天+** | 🔴 | 等待工资发放日/研究选股 |
| **全链路（点击→入金）** | **7-30天，长尾可达30天+** | 🔴 | |

### 4.2 Google Ads报告中的累计转化分布

| 点击后天数 | 累计转化归因比例 |
|:---|:---|
| 第1天 | 约30% 🔴 |
| 第3天 | 约50% 🔴 |
| 第7天 | 约75% 🔴 |
| 第14天 | 约90% 🔴 |

### 4.3 归因偏差的灾难性影响

**Google Ads默认将转化归因于点击发生的当天**：

```
错误操作（致命）：
看着"过去3天"数据说"CPA太高了，降价！"
↓
实质上是盲人摸象
↓
这3天点击产生的入金转化，要在20天后才会回传后台
↓
降价行为直接破坏了正在运行的出价学习
```

### 4.4 转化窗口设置建议

| 转化类型 | 建议转化窗口 |
|:---|:---|
| 开户申请（浅层） | 30天 |
| 首次入金（深层，推荐目标）| **60天** 🔴 |
| 评估账户整体ROI | **永远排除最近14-21天数据** |
| 快速识别高效时段 | 同时观察7天归因数据 |

## 五、Value-Based Bidding (VBB) 在券商领域的应用

### 5.1 核心价值差异（为什么需要VBB）

日本证券用户的LTV差异极大：

| 用户类型 | LTV估算 |
|:---|:---|
| 普通投资者（定投NISA，小额）| ~$200 🔴 |
| 活跃交易者 | ~$2,000 🔴 |
| 高净值（60岁+，首次入金1,000万日元）| ~$10,000+ 🔴 |

在前端开户时，这三类用户的点击行为**完全相同**，tCPA无法区分。VBB（tROAS）能引导算法去寻找后者。

**DeepSeek案例参考**（🟡 来源未明确）：某券商通过VBB使平均存款额提升**65%**，存款成本降低**15%**。

### 5.2 VBB应用的主要障碍

| 障碍 | 说明 |
|:---|:---|
| **价值定义难** | 一个25岁年轻人入金3万定投NISA，和60岁老人转入1,000万买高配당股，前端行为完全相同 |
| **隐私数据孤岛** | 日本《个人信息保护法》极严，券商极少愿意将"客户实际入金金额"通过API脱敏回传Google |

### 5.3 实战替代方案：静态计分法（Value Rules）

当无法回传真实入金金额时，采用**事件分级计分**喂养tROAS：

| 事件（转化类型） | 赋分 | 说明 |
|:---|:---|:---|
| 提交邮箱/注册（浅层） | 10分 | 最低价值 |
| 完成身份验证（eKYC完了） | 100分 | 有效意向用户 |
| **完成首次入金（初回入金完了）** | **1000分** | 真正高价值用户的强信号 |

**DeepSeek补充的Value Rules方案**：
```
规则设置：
"上传My Number卡完成" → +10%价值
"首次入金完了"        → +50%价值

效果：强迫VBB算法过滤低质填表客，
      寻找愿意入金的高净值潜客
```

## 六、Broad Match + Smart Bidding 在日本金融市场

### 6.1 日语金融词汇的"意图灾难"（Intent Drift）

日语存在大量同音异义词（同音異義語）和省略语，Broad Match在冷启动阶段极易匹配到垃圾流量：

| 投放关键词 | 可能匹配到的垃圾流量 |
|:---|:---|
| `株価 予想`（股价预测）| "投資詐欺 LINEグループ"（诈骗荐股群）、"競馬 予想"（赛马预测）|
| `NISA おすすめ` | `NISA やめとけ`（NISA千万别做，负面情绪词）|
| `投資 初心者` | 各类免费课程/诈骗课程 |

### 6.2 Broad Match的实际效果数据

| 指标 | Exact Match | Broad + Smart Bidding | 可信度 |
|:---|:---|:---|:---|
| CTR | 高 | 中 | 🔴 |
| CVR | 高 | 中高 | 🔴 |
| 点击量 | 基准 | +40%-80% | 🔴 |
| 转化数 | 基准 | +20%-50% | 🔴 |
| CPA | 最低 | +10%-20% | 🔴 |

**结论**：Broad Match能显著扩大流量规模，但CPA略高于Exact，CVR略低。核心价值在于**"在茫茫词海中寻找高净值股民"**。

### 6.3 Broad Match使用铁律（日本金融市场）

**冷启动期（Cold Start）绝对禁止使用Broad Match**，正确流程：

```
Step 1：先用 Exact/Phrase Match 跑满50-100个深层转化（入金）
         ↓
Step 2：建立庞大的日语否定关键词库（见下表），封死垃圾流量
         ↓
Step 3：开启 Broad Match + tCPA/tROAS
         此时算法已"吃饱"高质量转化特征，才能真正发挥威力
```

**日语否定关键词库（必须预先封死）**：

```
诈骗/负面类：
詐欺、無料、やめとけ、失敗、大損、怪しい、注意

竞猜/非投资类：
競馬、競艇、オートレース、パチンコ、スポーツくじ

学术/学习类（非开户意图）：
レポート、勉強、英語、本

其他：
チャート 見方（仅查看图表，非开户）
投資 本（买书，非开户）
```
