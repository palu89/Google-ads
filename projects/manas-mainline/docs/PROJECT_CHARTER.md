# 项目章程

## 项目定义

- 项目类型：历史穿越
- 方法要求：严肃历史模拟
- 时间范围：`1927-1953`
- 历史范围：中国近代史主线，允许追踪与之直接相连的国际外部影响

## 项目硬约束

- 任何具体历史人物、战役、政策、组织、制度问题，默认先真实检索。
- 输出必须区分：`[硬史实] / [高可信推断] / [低可信推断] / [架空改写]`。
- 变量必须渐进传播，不允许一步改写全国历史。
- 历史变化必须经过：`注入 -> 扎根 -> 扩散 -> 对抗 -> 放大 -> 改写`。
- 不写“无代价碾压式穿越”，不写超时代外挂碾压。
- 项目经验只写回本项目学习链，不回流其他项目。

## 任务处理顺序

1. 先判断当前任务属于 `研究 / 设定 / 人物 / 时间线 / 卷章 / 章节推进` 的哪一类。
2. 判断是否需要补真实资料。
3. 判断是否需要先读取 `历史基线` 与 `项目学习链`。
4. 判断当前更适合推进哪一批。
5. 审查是否存在 `历史锚点冲突 / 逻辑冲突 / 人物失真 / 变量失控`。
6. 若任务已进入具体文本产出，再转给创作线程执行。

## 信心标签标准

- `[硬史实]`：本轮已通过可信来源核到，可作为项目锚点引用。
- `[高可信推断]`：与已核史实、地理、组织结构、时代条件高度相容。
- `[低可信推断]`：目前只是可写候选，需要补专题来源。
- `[架空改写]`：从某个注入点开始，经因果链推动产生的主动偏离。

## 当前项目阶段判断

- 当前阶段：`基线搭建期`
- 当前优先级：`高于剧情草写`
- 当前首要目标：形成一套可以长期复用的项目控制平面

## 当前推荐方向

- `[高可信推断]` 最稳的主变量，不是“直接改变高层决策”，而是“改变基层组织效率、生存率、情报流速、交通补给与干部留存率”。
- `[高可信推断]` 最稳的样板开局，不是直接进入全国决战，而是进入 `1927-1931` 的断裂与扎根阶段。
- `[低可信推断]` 如果你偏爱更强戏剧性，也可把起点推到 `1936-1937`，但那会更快接触大人物与强锚点，难度更高。

## 禁止项

- 不把后世术语直接套入当年组织、制度与口语。
- 不把单一一次行动直接写成全国连锁改写。
- 不让真实历史人物无条件信任主角。
- 不让主角凭现代常识轻易解决年代性结构问题。
