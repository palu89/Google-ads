# 研究协议

## 适用范围

- 一切具体历史人物、战役、政策、组织、制度、称谓、建制、装备、交通、货币与地理问题。

## 强制检索情形

- 提到具体真实人物。
- 提到具体战役、会议、政策、组织或制度。
- 要求来源链接或考据说明。
- 准备把某个历史节点写进主线、细纲或章节。

## 来源优先级

1. 原始史料、档案、回忆录、当时报刊、制度文件。
2. 大学、研究机构、国家档案馆、博物馆。
3. 有注释体系的学术著作与论文。
4. 综合性权威百科或官方历史概述。
5. 二手整理材料，仅作线索，不直接落为锚点。

- 结论正文：

- 可直接写入的范围：
- 暂时不能写死的部分：
- 需二次核验的问题：

## 项目级写法要求

- 能写绝对日期，就不只写相对日期。
- 能写时代称谓，就不用后世泛称。
- 两个来源若存在冲突，先标冲突，不强行统一。
- 未检索的细节不得伪装成硬史实。

## 常见误差警报

- 把后来的组织建制提前到更早年份。
- 把后来的常用术语提前到更早语境。
- 把装备、外援、交通条件写得过于先进。
- 把地方社会结构写得过分整齐。
- 把真实人物的路线与动机写得过于单线。

## 与创作线程的接口

- 本线程负责：问题拆分、资料锚点、边界判断、可写范围。
- 创作线程负责：人物卡、分章、场景、对白、战斗、草稿。
- 若具体文本要落笔，本线程先给出“可写边界 + 必避误差 + 必核问题”。
